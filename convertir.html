    Arquitectura híbrida — el código detecta la estructura; la IA solo interpreta

    1. Fase 1 — Extracción local: se lee el DOCX completo y se extrae todo el texto sin IA, sin truncar, preservando el orden.
    2. Fase 2 — Parseo estructural local (sin IA): el código detecta bloques (BLOQUE N: …) y variables (Variable N: …) usando regex robustos. No usa IA para esto.
    3. Fase 3 — Datos duros locales (sin IA): el código extrae directamente del texto los puntajes D, P, M, Im, V, E, la suma de criterios, la fórmula y el puntaje final de cada variable.
    4. Fase 4 — IA solo por variable: para cada variable, 2 llamadas acotadas: una para resumen/fortalezas/debilidades/brechas y otra para las 10 secciones narrativas. La IA no descubre estructura ni puntajes.
    5. Fase 5 — Prioridad absoluta de datos locales: los puntajes extraídos por código nunca pueden ser modificados por la IA. Si el código extrajo D=2, el JSON final tendrá D=2.
    6. Fase 6 — Validación estricta: 8 bloques, 30 variables, todos los campos requeridos. Si algo falla, no se guarda nada y se muestra el error exacto.
    7. Documentos ya procesados (mismo hash SHA-256) no se reprocesarán — cero costo adicional.
    Formato requerido del DOCX: los bloques deben usar encabezados tipo BLOQUE 1: Nombre y las variables Variable 1: Nombre. Los puntajes deben seguir el patrón D: 2 / P: 1 / M: 1 / Im: 1 / V: 1 / E: 2 y la fórmula (8/12) × 10 = 6.67.
