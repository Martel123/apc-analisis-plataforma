IMPORTANTE:

1. crear o editar archivos JSON dentro de una carpeta llamada /analisis

1. OBJETIVO DEL SISTEMA
==================================================

Construir una plataforma interactiva para:
- analizar un candidato

- navegar por bloques temáticos

La app debe servir para ciudadanos, periodistas, analistas, investigadores y equipos técnicos.

4. METODOLOGÍA DEL SISTEMA
==================================================

La plataforma se basa en una metodología estricta de evaluación técnica.

Características:
- 30 variables de análisis
- 8 bloques temáticos
- 6 criterios por variable:
  1. Diagnóstico
  2. Propuesta
  3. Medidas
  4. Implementación
  5. Viabilidad
  6. Especificidad
- cada criterio se puntúa de 0 a 2
- puntaje final por variable:
  (suma de criterios / 12) * 10
- redondeo a 1 decimal
- solo se puntúa evidencia explícita
- debe poder mostrarse cuando hubo correcciones metodológicas
- debe existir espacio para notas metodológicas por variable y por candidato

6. PROCESAMIENTO AUTOMÁTICO DE DATOS
==================================================

Quiero que el sistema haga automáticamente lo siguiente al leer /analisis:

1. detectar todos los candidatos disponibles
2. construir el selector de candidatos
3. calcular ranking general por total_score
4. construir comparación por bloques
5. construir comparación por variables
6. construir heatmap general
7. construir KPIs automáticos:
   - mejor bloque
   - peor bloque
   - variable más fuerte
   - variable más débil
   - promedio global
   - consistencia
8. construir tarjetas de fortalezas y debilidades
9. detectar si una variable tiene corrected_methodology = true
10. mostrar notas metodológicas globales y específicas

También quiero una capa de procesamiento que:
- derive bloques automáticamente si vienen en los JSON
- normalice nombres
- ordene variables por puntaje
- genere comparaciones entre candidatos aunque el orden del JSON sea distinto
